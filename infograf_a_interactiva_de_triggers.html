¿Qué es un Trigger?

Automatización Reactiva en el Motor de Datos

Un Trigger (o disparador) es un objeto especial de programación dentro de SQL Server que se ejecuta de manera automática e inmediata cuando ocurre un evento específico de manipulación de datos (INSERT, UPDATE o DELETE) en una tabla objetivo.

"Garantizan la consistencia de la información y la ejecución de reglas de negocio complejas directamente en el servidor sin intervención del software cliente."

INSERT

Se dispara cuando se añade una nueva fila. Útil para rellenar campos por defecto, calcular totales o sincronizar réplicas.

UPDATE

Se ejecuta al modificar valores existentes. Permite contrastar el valor anterior con el nuevo para auditorías rigurosas.

DELETE

Se activa ante eliminaciones. Ideal para restringir borrados prohibidos o archivar información eliminada preventivamente.

El Motor Interno

Las Tablas Especiales: INSERTED y DELETED

Al ejecutarse un Trigger, SQL Server crea temporalmente dos tablas lógicas en memoria. Son la clave para analizar los cambios de estado de las tuplas.

INSERTED

Almacena las Nuevas Versiones

Contiene copias de las filas afectadas durante las instrucciones INSERT y UPDATE. Representa el estado final que los datos quieren tomar en la base de datos física.

Comportamiento según Acción:

  • INSERT: Guarda las nuevas tuplas insertadas.
  • UPDATE: Guarda los nuevos valores que reemplazarán a los antiguos.
  • DELETE: Permanece vacía.
DELETED

Almacena el Historial Anterior

Contiene copias de las filas que fueron borradas por un DELETE, o los valores antiguos que fueron reemplazados al ejecutar un UPDATE.

Comportamiento según Acción:

  • INSERT: Permanece vacía.
  • UPDATE: Guarda los valores previos a la modificación.
  • DELETE: Guarda las filas que acaban de ser removidas.
Caso de Estudio

Vigilancia de Ventas: trg_Auditoria_OrderDetails

Este disparador desarrollado para la base de datos Northwind ejerce control estricto sobre la tabla Order Details (Detalles de los Pedidos).

Si un operario altera la cantidad de un producto, asigna un descuento alternativo o elimina un ítem de un pedido previamente registrado, el trigger interviene inmediatamente para registrar el suceso histórico de forma inalterable.

Flujo de Auditoría

Tabla Detalle ➔ Trg_Auditoria ➔ Tabla Auditoría
Implementación en T-SQL 
CREATE TRIGGER trg_Auditoria_OrderDetails
ON [Order Details]
AFTER UPDATE, DELETE
AS
BEGIN
    -- Registrar cambios de datos en la tabla de Auditoría
    INSERT INTO AuditoriaOrderDetails (
        Accion, OrderID, ProductID, UnitPrice, 
        Quantity, Discount, FechaAuditoria, Usuario
    )
    SELECT 
        CASE WHEN EXISTS(SELECT 1 FROM inserted) THEN 'UPDATE' ELSE 'DELETE' END,
        d.OrderID, d.ProductID, d.UnitPrice, 
        d.Quantity, d.Discount, GETDATE(), SYSTEM_USER
    FROM deleted d
END

Conclusiones Técnicas del Taller

1. Automatización e Integridad

Los triggers permiten centralizar reglas de negocio directamente en el motor de la BD, reduciendo la redundancia de código en las aplicaciones clientes y asegurando que las directivas se ejecuten sin excepción.

2. Desempeño y Auditoría Inalterable

A través del uso de las tablas transaccionales en memoria, se logran desarrollar logs históricos eficientes capaces de rastrear al usuario del sistema operativo y la marca de tiempo exacta de cualquier cambio malintencionado.
